Lists

When to Use

Lists are great for presenting a series of related phrases or sentences that need to be read faster.

Which to Use

Foundation styles allow for a multitude of lists, including: ul for generic lists and ol for numbered lists.

How to Use

Note the code snippets under each subheading on the left; look there for required classes and list markup.

Editing the CSS

CSS for list styles can be found in _settings.scss and the variables are $list-*.

HTML 

<ul class="inline">
  <li>
    <a href="#">inline-list element one</a>
  </li>
  <li>
    <a href="#">inline-list element two</a>
  </li>
  <li>
    <a href="#">inline-list element three</a>
  </li>
</ul>

Forms

Elements of a Form

A form should be marked up using it's default HTML properties. The ones we make use of include (in hierarchical order):

  • form
  • label
  • input
  • select
  • textarea
  • button

How to Use

Make forms great and easy to use with the following rules:

  1. Wrap checkboxes and radio buttons within labels for larger hit areas, and be sure to set the for, name, and id attributes for all applicable elements.
  2. Series of checkboxes and radio buttons below within a
      <ul class="(inline) actions">
  3. Before selecting any set of fields to use for a required input, explore other options (e.g., radio buttons over select lists).

HTML 

<form>
  <div class="large-12 columns">
    <label>
      Input Label
      <input placeholder="large-12.columns" type="text">
    </label>
  </div>
</form>
